107  @brief Attempts to read a byte from the bluetooth connection until a full
108  packet is recieved.
109  @details This must be called repeatedly until it returns true. Packet data
110  will not start being stored until PACKET_START is read, at which point
111  the buffer will begin to be populated. Once the full packet is recieved
112  and the checksum is valid, this function will overwrite lastPacket
113  (getLastPacket()) and return 1 ONCE. If you call it again, it will try
114  looking for another packet BUT WILL NOT overwrite the last packet until

116  @return 1 if a full packet has been read and checked, 0 otherwise
117 */
